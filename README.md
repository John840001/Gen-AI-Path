## **2. Strategy — Why This Path** 

- Skip re-learning DSA/DBMS/cloud from scratch — you already have this. Time goes straight into AI-specific skills. 

- Specialize early in GenAI (LLM APIs, RAG, agents) rather than broad ML theory — this is the highestdemand, fastest-converting track for someone with your background. 

- Build 3 portfolio projects, not just certificates — interviewers weight a live, deployed project far higher than a badge. 

- Layer one or two paid certifications on top of free learning once your portfolio exists — certs support the story, they don't replace it. 

- Start applying in parallel from Month 4 onward — don't wait for the plan to be 'finished' before you start 

   - interviewing.

## **6. Interview Preparation Checklist** 

- 100–150 medium-difficulty coding problems (LeetCode or equivalent) — your DSA base is already there, this is refresh + speed. 

- ML/GenAI fundamentals: supervised vs. unsupervised learning, evaluation metrics, transformer architecture and attention at a conceptual level. 

- RAG architecture trade-offs: chunking strategies, embedding model choice, vector DB selection criteria. 

- Prompt engineering techniques and basic LLM evaluation/guardrail methods. 

- System design for GenAI: 'design a RAG pipeline at scale', 'design an AI customer-support agent', latency/cost/caching trade-offs. 

- Full-stack + AI integration patterns: function calling, streaming responses, error handling around AI API calls, client- vs. server-side LLM calls. 

- 4–5 STAR-format behavioral stories drawn from your existing IT experience, reframed around ownership and impact. 

- Be ready to walk through all three portfolio projects end-to-end, including what broke and how you fixed it.

## **8. How to Use This Plan** 

- Re-evaluate at the end of every month — if a phase is taking longer, borrow time from the following week's buffer rather than skipping fundamentals. 

- If GenAI applications aren't converting to interviews by Month 5, broaden to MLOps or AI-platform roles in parallel — the skill overlap (cloud, APIs, deployment) is high, so the pivot costs little extra prep time. 

- Keep every project link, certificate, and interview note in one place (this doc or a linked tracker) so your story stays consistent across applications. 

- Protect the weekend blocks — they're where the deep project work happens; weekday hours are best spent on courses, reading, and small coding tasks.
